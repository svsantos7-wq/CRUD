# Projeto CRUD em PHP - Atividade Prática de Desenvolvimento Web

## Sobre o projeto

Este repositório contém um projeto simples de CRUD em PHP com conexão ao banco de dados utilizando PDO.

O projeto foi construído com foco didático e aborda conceitos importantes para quem está começando ou consolidando conhecimentos em PHP e Desenvolvimento Web, como:

- organização básica de arquivos PHP;
- conexão com banco de dados;
- uso de `PDO`;
- consultas com `SELECT`, `INSERT`, `UPDATE` e `DELETE`;
- tratamento de dados enviados por formulários;
- validação básica;
- listagem dinâmica de dados;
- edição e exclusão de registros;
- introdução à orientação a objetos com a classe `User`.

Além disso, foi criado um **README específico com a explicação de cada arquivo do projeto**, para auxiliar no entendimento da estrutura e da responsabilidade de cada parte do sistema.

## Objetivo da atividade

Esta atividade tem dois grandes objetivos:

### 1. Compreender os conceitos PHP trabalhados no projeto
Vocês devem analisar o código e compreender os conceitos que estão sendo utilizados ao longo da aplicação.

Entre os principais conceitos abordados estão:

- `require` e organização de arquivos;
- uso de classes;
- conexão com banco via `PDO`;
- `prepare()` e `execute()`;
- captura de dados com `$_POST` e `$_GET`;
- `filter_input()`;
- validação de dados;
- uso de formulários HTML integrados com PHP;
- exibição dinâmica de dados vindos do banco;
- noções iniciais de orientação a objetos;
- separação de responsabilidades com a classe `User`.

### 2. Melhorar o layout e a experiência visual do projeto
Como esta disciplina é de **Desenvolvimento Web**, o objetivo não é apenas entender o PHP, mas também melhorar a apresentação da aplicação.

Vocês deverão propor uma melhoria visual para o sistema, tornando-o mais agradável, organizado e com melhor experiência de uso.

## Fluxo esperado no GitHub

### 1. Fazer o fork
Crie uma cópia deste repositório na sua conta GitHub.

### 2. Clonar o fork
Clone o seu fork na sua máquina.

### 3. Criar ou organizar suas alterações
Faça as melhorias no projeto.

### 4. Commitar
Registre suas alterações com mensagens claras.

Exemplos:
- `feat: melhora o layout da página inicial`
- `fix: corrige nome de campo no formulário`
- `style: reorganiza tabela e botões`
- `refactor: melhora estrutura do formulário`

### 5. Enviar para o GitHub
Faça o `push` para o seu fork.

### 6. Abrir o Pull Request
Abra um Pull Request para que as alterações possam ser analisadas.
